# Knowledge Codex — 設計書

制作中に得た知識をコレクションし、AIに整理させて、自分専用の制作知識DBに育てるアプリ。
主な利用者は作者本人（絵描き出身・CG/コーディング学習中）。転職ポートフォリオを兼ねる。

## 方針

- 便利さ最優先。サイバー装飾（黒〜濃紺＋シアン発光、ホログラムパネル）はその上に載せる
- ゲーム化しない。コレクション感は「ID採番・カウンタ・用語集・使用済バッジ」で出す
- Web版＝実用本体（このリポジトリ）。UE版＝同じJSONを読む閲覧専用のポートフォリオデモ（別途）

## データ形式（すべての土台）

Web版・UE版・AI整理はすべてこのJSONを読み書きする。

```json
{
  "id": "KX-0042",
  "title": "水面の動きがゴムっぽく見える原因",
  "oneLiner": "水っぽさは高さ変位でなく法線・反射・リップルで出す",
  "body": "自分のメモ原文",
  "attachments": ["img_0042_1.png"],
  "links": ["https://..."],
  "category": "CG/水表現",
  "tags": ["WPO", "Normal", "Water"],
  "type": "solution",
  "importance": 2,
  "status": "enriched",
  "favorite": true,
  "ai": {
    "summary": "", "keyPoints": [], "terms": [{"term": "", "desc": ""}],
    "useCases": [], "nextKeywords": [], "nextExperiments": [],
    "suggestedTags": [], "suggestedCategory": "", "relatedIds": []
  },
  "appendLog": [{"date": "2026-07-04", "text": ""}],
  "usedIn": [{"project": "", "date": "", "note": ""}],
  "createdAt": "", "updatedAt": ""
}
```

- `type`: `solution` / `insight` / `term` / `recipe` / `prompt` / `reference`
- `ai` は自分のメモと分離して保持（混ぜない）
- 追記は本文編集でなく `appendLog`（日付つき追記ログ）
- ID `KX-0042` の連番がコレクション感の核

## ステータス（4状態＋フラグ）

| 状態 | 意味 | 表示 |
|---|---|---|
| `captured` | 登録しただけ・整理待ち | 琥珀チップ「未整理」 |
| `enriched` | AI整理を採用済み | シアンチップ「AI整理済」 |
| `applied` | 制作で使用済み | 緑チップ「使用済」 |
| `archived` | 保管（一覧でデフォルト非表示） | 灰色 |

お気に入り・関連リンク有無・理解度は状態でなくフラグ/自動判定。7段階の状態管理はしない。

## カテゴリ・タグ

- カテゴリ＝1つだけ選ぶ棚。大分類6つ（`CG` / `UI・UX` / `Design` / `Coding` / `AI・Prompt` / `Workflow`）＋自由な中分類（例 `CG/水表現`）
- タグ＝何枚でも貼るラベル（Unreal Engine, Shader, Lighting などはタグ）
- タグ荒廃対策: 入力は既存タグのサジェスト優先／AIのタグ案は既存タグ一覧をプロンプトに含めて既存優先／リネーム・統合はv2

## 画面（実質7画面）

一覧・検索・お気に入り・最近追加は「一覧画面のフィルタプリセット」に統合。フィルタ状態はURLハッシュに持つ。

1. **ホーム**: 検索窓（`/`でフォーカス）＋CAPTUREボタン＋最近の知識＋未整理カウンタ＋カテゴリ別件数＋最近覚えた技術語。ヘッダに `RECORDS: n // SYNCED` のモノスペース表示
2. **登録**: 必須はタイトルのみ。タイトル＋一言メモ＋[保存/保存してAI整理]。詳細（本文・画像Ctrl+V・リンク・カテゴリ・タグ）は折りたたみ。保存後は同画面に留まり連続登録
3. **AI整理**: 左=元メモ、右=AI提案。提案は点線ボーダー→採用で実線。タグ案はチップをタップで追加。「次に調べる」各項目に「新規知識として登録」ボタン
4. **一覧**: 左レールにフィルタ（カテゴリ/状態/タイプ/重要度/★）、カードグリッド。カード=サムネ（無ければカテゴリアイコン）+ID+タイトル+一言+タグ3個+状態チップ+◆重要度+★+更新日
5. **詳細**: ヘッダ（ID/タイトル/状態/★/操作）→ 3カラム（添付｜自分のメモ=主役｜AIパネル）→ 下段（追記ログ/関連知識/使用履歴）。モバイルはタブ切替。「使用済にする」はミニダイアログでプロジェクト名を聞いて `usedIn` に追記
6. ネットワークグラフ（v3以降）
7. 設定: JSONエクスポート/インポート（最重要）、APIキー、表示密度。チュートリアルは初回オーバーレイ3枚＋サンプル知識3件のプリセット

## AI整理の実装方式

GitHub Pages（静的・サーバーなし）前提。

- **MVP**: プロンプトコピペ方式。「整理プロンプトをコピー」→ Claude/ChatGPTに貼る → 返ってきたJSONを貼り付け欄にペースト → 構造化して取り込み
- **v2**: 設定画面で自分のAnthropic APIキーをlocalStorageに保存し、ブラウザから直接呼び出し（`anthropic-dangerous-direct-browser-access` ヘッダ）

## 実装フェーズ

- **Phase 1（MVP核）**: スキーマ確定 → 登録（タイトル+一言+保存）→ 一覧（カード+フィルタ+検索）→ 詳細 → IndexedDB → JSONエクスポート/インポート
- **Phase 2（AIループ)**: AI整理コピペ方式 → 提案採用UI → ステータス → 追記ログ → 使用済にする
- **Phase 3（体験強化）**: ホームのホログラム演出 → 画像添付 → 用語集 → テーブル表示 → タグ統合管理
- **Phase 4（発展)**: ネットワークグラフ → API直接連携 → UE版ビューア

**最初に作らないもの**: 派手な演出・イラスト生成・カードゲーム要素・レベル/XP・ネットワークグラフ・ログイン/同期（JSONエクスポートで代替）

## ポートフォリオデモ（完成後）

1本のストーリーで見せる: 詰まる → 30秒で登録 → AI整理で技術語・関連知識が補足 → 後日別プロジェクトで検索 → 再利用 → 使用済バッジ。最後にUE版で同じDBが3D空間に浮かぶカット。
